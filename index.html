ByteStringとText

Stringは非効率

Haskellの文字列は、標準ではStringの値として扱われます。 しかしStringは、特に長い文字列については、速度的にもメモリ的にも非効率です。

そのために、より効率良く文字列データを扱うために

  • Data.ByteString(ASCII文字用)
  • Data.Text(Unicode文字用)

が用意されています。

なんかいっぱいある…

実際にモジュールを見ると、用途によって、下記のようにいくつかの型が用意されています。

正格 遅延
Word8で管理 Data.ByteString Data.ByteString.Lazy
Charで管理 Data.ByteString.Char8 Data.ByteString.Lazy.Char8
Text Data.Text Data.Text.Lazy

Word8とCharの違い

関数の引数/戻り値の型が、Word8となるかCharとなるかの違いです。

例として、まずは下記のプログラムを見てみます。

import Prelude hiding (getLine, putStr)
import Data.ByteString

main :: IO ()
main = do
    ln <- getLine
    putStr ln
